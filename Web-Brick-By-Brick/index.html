Brick by Brick

web easy 100 pts · UMass CTF 2026

The challenge description says there's a hidden admin dashboard on an old portal for BrickWorks Co. and asks us to find it and get the flag. The instance is running at brick-by-brick.web.ctf.umasscybersec.org:32769.

01 Initial Recon

Visiting the root page shows a generic maintenance message — nothing interactive. No forms, no JS requests to interesting endpoints, nothing in the page source worth noting. 

http://brick-by-brick.web.ctf.umasscybersec.org:32769/
→ "Internal systems are currently undergoing maintenance."

First obvious thing to check on any web challenge — robots.txt.

02 robots.txt — Hidden Paths

User-agent: *
Disallow: /internal-docs/assembly-guide.txt
Disallow: /internal-docs/it-onboarding.txt
Disallow: /internal-docs/q3-report.txt

# NOTE: Maintenance in progress.
# Unauthorized crawling of /internal-docs/ is prohibited.

Three internal documents listed. assembly-guide.txt and q3-report.txt were dead ends — generic company content. The useful one was it-onboarding.txt.

ℹ robots.txt is not a security mechanism — it's a public roadmap. Developers often list sensitive paths there assuming bots won't visit them, but anyone can read the file.

03 LFI via ?file= Parameter

The IT onboarding document revealed two key things:

Finding 1 — File Parameter

The internal portal accepts a ?file= GET parameter to serve files. No mention of validation or access control.

Finding 2 — Config Location

Admin credentials are stored in config.php in the web root. The IT doc explicitly tells staff to look there.

Combining both — we can read config.php directly through the exposed parameter:

http://brick-by-brick.web.ctf.umasscybersec.org:32769/?file=config.php

The server responds with the raw PHP source, no authentication required:

// BrickWorks Co. — Application Configuration
// WARNING: Do not expose this file publicly!

// The admin dashboard is located at /dashboard-admin.php

// WARNING: SYSTEM IS CURRENTLY USING DEFAULT FACTORY CREDENTIALS.
// TODO: Change 'administrator' account from default password.

define('ADMIN_USER', 'administrator');
define('ADMIN_PASS', '[deleted it for safety reasons - Tom]');
⚠ The config warns that default factory credentials are still in use. Tom removed the password from the comment but never actually changed it. The default password turns out to be administrator.

04 Admin Panel

The config also gives us the admin panel path: /dashboard-admin.php. Logging in with administrator / administrator works immediately. 

http://brick-by-brick.web.ctf.umasscybersec.org:32769/dashboard-admin.php
credentials: administrator / administrator
→ CHALLENGE COMPLETE — FLAG CAPTURED

// takeaways
  • robots.txt is public — never list sensitive paths there
  • File inclusion parameters need strict allowlist validation
  • Config files with credentials must never be in the web root
  • Always change default credentials before deploying to production
